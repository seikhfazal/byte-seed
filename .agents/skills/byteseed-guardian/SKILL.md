# ByteSeed Guardian

Protect the stable educational model and repository before accepting any ByteSeed change.

## Establish the baseline

1. Confirm the requested branch and inspect `git status --short` before writing.
2. Read the affected source, configuration, documentation, and existing regression workflow.
3. Treat `checkpoints/anchor_v2_3_finetuned.pt` as the current stable local checkpoint unless repository documentation explicitly changes that policy.
4. Preserve existing checkpoint loading, tokenizer compatibility, CLI defaults, and manual model behavior unless the task explicitly authorizes a breaking change.

## Protect artifacts and data

- Never create, overwrite, rename, delete, stage, or push checkpoints without explicit authorization.
- Never train, pretrain, run SFT, rebuild data, train the tokenizer, download datasets, or modify `data/`, `tokenizer/`, `checkpoints/`, or `runs/` unless explicitly requested.
- Inspect only trusted locally generated ByteSeed checkpoints; prefer `map_location="cpu"` for metadata.
- Keep `.venv/`, checkpoints, processed data, runs, tokenizer binaries, imported data, logs, and secrets excluded according to `.gitignore` and `docs/REPO_HYGIENE.md`.
- Scan proposed public changes for secrets and personal data using `docs/PUBLIC_RELEASE_CHECKLIST.md`; review every match in context.

## Control commands

Allow read-only inspection, compilation, conventional unit tests, parameter counting, dataset inspection, fixed regression evaluation, short inference benchmarks, and Git status/diff checks when they do not write protected artifacts. Reject or pause commands that perform optimizer steps, regenerate data, alter tokenizer files, write checkpoints, download content, push, merge, tag, or release unless the user explicitly expands scope.

## Review changes

1. Inspect `git diff --check`, `git diff`, and `git status --short`.
2. Verify generated artifacts remain ignored and unstaged.
3. Run the smallest relevant deterministic checks, then the stable anchor-retention regression when runtime behavior changes.
4. Distinguish anchor retention from held-out generalization; never infer the latter from training-overlapping prompts.
5. Require backward-compatible defaults for checkpoint schemas, tokenizer handling, generation, and CLI behavior.
6. Preserve manual attention as the educational reference. Permit SDPA, KV caching, compilation, or other optimized paths only as explicitly approved optional paths with parity tests and legacy defaults unchanged.

## Expected output

Report the baseline, allowed actions taken, prohibited actions avoided, validation results, compatibility risks, artifact/Git status, and any blocker. Do not accept a change without reviewing its diff and explaining residual risk.
